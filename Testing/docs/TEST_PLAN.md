# SpotMe API — Test Plan

## 1. Scope

Covers the SpotMe server API (`server/`) running on Node.js/Express/SQLite.
Excludes: frontend (React), Groq AI endpoints (require paid API key), OAuth flow (requires live Google credentials).

### In-Scope Endpoints
| Endpoint | Methods |
|---|---|
| `/api/health` | GET |
| `/api/auth/signup` | POST |
| `/api/auth/login` | POST |
| `/api/auth/logout` | POST |
| `/api/auth/me` | GET |
| `/api/auth/account` | DELETE |
| `/api/sessions` | GET |
| `/api/sessions/:id` | GET, PATCH, DELETE |
| `/api/profile` | PATCH |
| `/api/profile/upgrade` | POST |

---

## 2. Objectives

1. Verify all API endpoints return correct status codes and response shapes.
2. Confirm auth middleware rejects unauthorized requests.
3. Validate signup/login input validation logic (EP, BVA, decision tables).
4. Confirm user data isolation (user A cannot see user B's sessions).
5. Measure API performance under normal (50 VU) and stress (200 VU) conditions.
6. Identify breaking points under extreme load.

---

## 3. Test Approach

### A. Black-Box Functional Testing
- **Tool**: Vitest + Supertest (in-memory sql.js DB)
- **Techniques**: EP, BVA, Decision Tables, State-Transition, Use-Case Scenarios
- **Files**: `tests/integration/auth.test.js`, `tests/integration/sessions.test.js`

### B. White-Box Unit Testing
- **Tool**: Vitest
- **Coverage target**: ≥ 80% statement coverage on `routes/` and `_shared.js`
- **File**: `tests/unit/shared.test.js`

### C. Performance Testing
- **Tool**: k6
- **Files**: `tests/k6/smoke.js`, `tests/k6/load.js`, `tests/k6/stress.js`

---

## 4. Test Environment

| Component | Value |
|---|---|
| Runtime | Node.js ≥ 18 |
| OS | Windows 11 / Linux |
| DB | sql.js (WASM in-memory for unit/integration; real SQLite for perf) |
| Port (perf tests) | 8787 |
| k6 version | ≥ 0.47 |
| Vitest version | ≥ 1.0 |

### Setup
```bash
# Install deps (inside server/)
npm install

# Run unit + integration tests
npm test

# Run with coverage
npm run test:coverage

# Performance (requires running server on port 8787)
npm run test:k6:smoke
npm run test:k6:load
npm run test:k6:stress
```

## 8. Risks & Mitigations

| Risk | Mitigation |
|---|---|
| sql.js WASM init slow in CI | Cache WASM binary; use `beforeAll` init |
| bcrypt makes signup slow under load | Expected; tune concurrency limits |
| Groq endpoints not tested | Covered by manual testing; mock in future |
| Port 8787 conflict during perf tests | Kill existing process before test run |
